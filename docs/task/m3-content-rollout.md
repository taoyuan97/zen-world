## 2. 验收标准

- [ ] 10 座山全部可完整走通：进山 → 对话 → 冥想（5 与 10 分钟两档）→ 点亮 → 回地图点亮
- [ ] 9 个新主题视觉差异明显：配色、装饰、天空时段（day/night/sunset）、粒子均按配置生效
- [ ] 10 位老师外观可区分（配色 + 配饰），待机动作正常
- [ ] 全部 JSON 通过校验器；任意文件损坏不影响其他山
- [ ] 点亮 10/10 后地图全景氛围符合预期（各山灯色/萤火共存不杂乱）
- [ ] 文案校对完成，无占位文本残留
- [ ] 性能复查：任一场景 60fps、draw calls < 80
- [ ] `npm run build` 通过

## 3. 已确认决策（2026-07-20 用户确认）

| # | 决策点 | 结论 |
|---|---|---|
| D1 | 文案工作流 | 沿用 M2 D3/D4：每山先出 md 存档（`docs/material/dialogue/<hill>.md`、`docs/material/guide-words/<hill>-{5,10}min.md`）再生成对应 JSON；27 份一次性起草完成，用户后续集中校对（任务 3.8） |
| D2 | 装饰/粒子/星空路线 | 全部程序化生成（延续 M2 D2，InstancedMesh + 共享材质，零许可风险）；水面反光片、沙丘曲线作为新 decor key（`water_glint`、`dune`）补入 hills.json；雾谷浓雾通过 `HillEnv` 新增可选 `fogNear`/`fogFar` 实现（缺省 26/85） |
| D3 | 老师配饰 | 10 种 prop 全程序化小件，不补充新 GLB；GLB 无手部骨点，配饰按固定偏移摆放（身侧/身前/胸前），不做手部挂接 |
| D4 | B3 呼吸节奏差异化 | 做最小版：仅温泉谷 5s/5s，其余 9 山维持 4s/4s；引导词 cue 节奏与之匹配（breathCycle 本为按脚本配置，零代码改动） |
| D5 | 机位 3（冥想中） | 全主题统一 2°/s 缓慢环绕，night/sunset 不加分支 |
| D6 | 执行与验收 | 顺序：视觉代码 → 文案一次性起草 → 全量校验 → 联调/十山遍历；产出 `docs/test/m3-acceptance.md` + 遍历测试记录；视觉差异与文案校对由用户人工验收（同 M2 模式） |

## 4. 风险与备注

- **工作量大头是文案**（3.4 + 3.5 约 4 人日起草 + 校对）：AI 起草后务必逐山朗读校对，尤其引导词的停顿节奏与 cue 时间轴匹配。
- 9 主题共用代码、只换配置；如发现某主题需要新代码（如水面反射），先评估能否用配置组合实现，避免破例。
- 星空原（night）与古寺（sunset）首次引入非 day 天空，skyDome 预设需在 3.2 一并落地。
