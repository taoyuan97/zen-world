## 1. 游戏概述

### 1.1 一句话概念

一款低多边形禅意风的 3D 冥想收集游戏：探索十座小山，与山中冥想老师相遇，完成 5~10 分钟引导冥想，逐一点亮整片禅境。

### 1.2 设计愿景

- 做"5~10 分钟一次的数字避静"：打开浏览器就能进入一段安静时间。
- **无失败、无竞争、无惩罚**。唯一的进度是内心的平静与地图上逐渐亮起的灯火。
- 核心情绪曲线：安静进入 → 被温柔引导 → 仪式感收尾 → 可见的完成感。

### 1.3 体验支柱（Experience Pillars）

1. **宁静探索** — 缓慢、柔和的世界浏览，没有催促。
2. **温柔引导** — 老师用简短对话把玩家带入状态，冥想过程有清晰的文字引导。
3. **可见的成长** — 每完成一次冥想，一座山被永久点亮；点亮全部 10 座山是长期目标。

### 1.4 目标用户与使用场景

- 想尝试冥想但难以坚持的人；需要工作间隙 5~10 分钟放松的人；睡前舒缓场景。
- 单次会话时长约 8~15 分钟（对话 1~3 分钟 + 冥想 5~10 分钟）。

### 1.5 平台与技术边界

- 桌面浏览器（Chrome / Edge / Safari 最新版），鼠标交互。
- 不依赖后端；进度存 localStorage；全部素材本地打包或运行时加载。
- 移动端触摸适配（OrbitControls 触摸、UI 响应式）在 M4 之后评估。

## 2. 核心玩法循环

```
┌─────────────────────────────────────────────────────┐
│  主场景：禅境地图（10 座小山，可旋转/缩放浏览）          │
│     │ 点击一座山（悬停高亮 + 名称浮签）                 │
│     ▼                                                 │
│  冥想场景（该山主题环境，相机运镜入场）                  │
│     │ 点击冥想老师                                     │
│     ▼                                                 │
│  对话（2~4 轮，打字机文字，含轻选项）                   │
│     │ 对话结束，选择时长：5 分钟 / 10 分钟              │
│     ▼                                                 │
│  冥想会话（文字引导 + 计时器 + 呼吸提示动画）            │
│     │ 计时结束                                         │
│     ▼                                                 │
│  点亮演出（光柱 + 粒子 + 色彩恢复） → 写入存档           │
│     │ 返回地图                                         │
│     ▼                                                 │
│  该山呈现点亮状态（灯火 / 萤火 / 色彩）                  │
└─────────────────────────────────────────────────────┘
长期目标：点亮全部 10 座山 → 触发全局完成演出（M4）
```

**规则细节**

- 已点亮的山可重复进入、重复冥想；重复完成不改变点亮状态，但累计冥想次数与时长。
- 冥想中途退出：本次不计完成，无惩罚，返回地图（配温柔提示文案，如"山会一直等你"）。
- 首次进入游戏显示开始页（标题 + 开始/继续）；有存档时展示已点亮进度 x/10。

## 3. 世界与场景设计

### 3.1 主场景：禅境地图

- 一张低多边形地形：中央山谷 + 平缓水面/云海，10 座小山散布其间，间距保证互不遮挡且点击区域充足。
- **未点亮状态**：低饱和配色、薄雾笼罩、山体无光源。
- **点亮状态**：山体暖色灯火（点光源 + 自发光灯笼/小屋）、雾气散开、常驻萤火/花瓣粒子、整体色彩恢复饱和。
- 相机：OrbitControls，限制俯仰角（约 20°~60°）与缩放范围，阻尼平滑；默认缓慢自动旋转，交互时暂停。
- 交互：Raycast 拾取；悬停时山体描边高亮 + 山名浮签（HTML）；点击进入场景（短淡出过渡）。
- 地图 HUD：右上角进度"已点亮 x/10"、静音开关；左下角操作提示。

### 3.3 冥想场景结构（共用模板）

所有冥想场景使用同一个 `MeditationScene` 模板，通过主题配置换肤：

- **空间布局**：一小块主题地形（直径约 40m），中央空地摆放蒲团/岩石座位，老师面向玩家入口方向。
- **三段机位**：
  1. 入场运镜（2~3 秒缓推，展示环境全景）；
  2. 对话机位（过肩视角，老师中景）；
  3. 冥想机位（缓慢环绕或固定远景，画面尽量不打扰）。
- **主题配置项**：地形装饰物（树木/岩石/建筑）、天空颜色/时间、雾密度与颜色、主光源色温、装饰粒子、配色板、环境音（M4）、老师配色与配饰。

## 4. 角色设计

- **MVP 方案（2026-07-19 已确认）**：选用 CC0 低模人物成品模型（GLB，如 Quaternius 系列），统一缩放/朝向与盘坐姿态，通过配色、斗笠/披风/手持物（竹杖、念珠、茶碗等）区分 10 位老师；程序化叠加待机动画：呼吸起伏、偶尔点头、手部微动。
- **进阶方案（M4 评估）**：Mixamo 免费骨骼动画（idle / 交谈手势）。
- 性格完全通过对话文本塑造：每位老师 1 个口头禅 + 1 种说话节奏（如竹心简短、星野诗意、暖阳热情）。
- 玩家角色：第一人称/无实体视角，不建模，降低工作量。

## 5. 系统设计

### 5.1 场景管理 SceneManager

- 两个顶级场景：`MapScene`、`MeditationScene`，统一 `dispose()` 释放 GPU 资源。
- 切换时 0.5s 黑场淡入淡出；`MeditationScene` 接收参数 `hillId`，从配置表加载主题。
- 全局单例：渲染器、时钟、存档、音频管理器；场景只持有自己的 Three.js 对象。

### 5.2 点亮与进度系统

- 状态机：`unlit → lighting（演出中，不可交互）→ lit`。
- 点亮判定：冥想会话计时完整走完即完成。
- 地图与冥想场景共用同一份点亮状态，返回地图时按状态重新着色/开灯。

### 5.3 对话系统

- **数据格式**：JSON 剧本，线性为主，支持轻选项（选项只影响下一句回复，不影响流程与结局），格式见附录 A。
- **表现**：底部对话框（HTML 浮层），打字机逐字显示，点击/空格继续；选项渲染为按钮。
- **节奏**：每位老师 2~4 轮对话，结尾自然过渡到"选择冥想时长"。

### 5.4 冥想会话系统（MVP：文字引导 + 计时器）

- **时长选择**：5 分钟 / 10 分钟两档；同一引导脚本的两种时间轴版本（附录 B）。
- **文字引导**：按时间轴触发的引导语 cue，屏幕中央下方淡入淡出，文案语气安静、短句。
- **计时器**：极简圆环进度 + 剩余分钟数；开始 10 秒后可点击"隐藏界面"。
- **呼吸辅助**：轻量圆环缩放动画（吸气放大 4s / 呼气缩小 4s，可在脚本 cue 中调整节奏）。
- **结束**：画面渐亮 → 播放点亮演出 → 完成页（本次时长、累计次数/时长、一句老师的赠言）。
- **中断**：Esc / 返回按钮弹出确认，确认退出则本次不计完成。

### 5.5 点亮演出 LightingRitual

- **场景内（约 4 秒）**：钟声（M4 前用 WebAudio 合成简音）→ 老师身后升起柔和光柱 → 金色粒子向四周扩散 → 场景配色从低饱和渐变到点亮配色。
- **地图**：对应山体的灯光开启、灯笼/小屋自发光、雾散、常驻萤火粒子；从高空俯瞰能明显区分亮/灭。

### 5.6 UI 系统（HTML/CSS 浮层，非 3D 内嵌）

| 界面 | 内容 |
|---|---|
| 开始页 | 标题、开始/继续按钮、进度 x/10 |
| 地图 HUD | 进度、静音开关、操作提示 |
| 场景 HUD | 返回地图按钮、对话框、时长选择面板 |
| 冥想 HUD | 计时圆环、引导文字、隐藏界面按钮 |
| 完成页 | 本次时长、累计统计、赠言、返回地图 |

- 视觉：大量留白、衬线中文（如 Noto Serif SC）、半透明深色底、圆角细边框。

### 5.7 音频系统

- **M1~M3（MVP）**：可无音频开发，或用 WebAudio 合成简单底噪/钵音占位。
- **M4**：Howler.js 接入环境音循环（每主题一条：竹林风声、湖面水声等）、交互音效、**引导语 TTS 语音**（与文字 cue 同一时间轴播放）。

### 5.8 存档系统

- localStorage，键名 `zen-world-save-v1`，结构见附录 C。
- 写入时机：点亮完成、设置变更；读取时机：启动。
- 版本字段 `version` 预留迁移能力。

## 6. 美术与音频规范

- **风格**：低多边形 + 平涂/轻微渐变 + 柔和雾效；避免写实贴图，全部使用顶点色或小平面色块。
- **调色板**：每主题 1 主色 + 2 辅色（见 3.2 表），未点亮状态整体降饱和 40%。
- **素材来源**：Quaternius / Kenney / Poly Pizza（均 CC0）；地形程序化生成（高度图 + 低模平滑）。
- **性能预算**：单场景三角面 < 50k，纹理总量 < 4MB，桌面中端显卡 60fps；装饰物实例化（InstancedMesh）。
- **音频（M4）**：环境音 CC0（Freesound 等），循环无缝；引导 TTS 选温和中低音色。

## 7. 技术架构

### 7.1 技术栈

| 层 | 选型 |
|---|---|
| 3D 引擎 | Three.js（OrbitControls、Raycaster） |
| 构建 | Vite + TypeScript |
| UI | 原生 HTML/CSS 浮层（打字机、计时器、面板） |
| 音频 | Howler.js（M4）；MVP 可 WebAudio 占位 |
| 存档 | localStorage |
| 素材 | CC0 低模（GLTF/GLB）+ 程序化地形 |

### 7.2 目录结构（建议）

```
zen-world/
├── index.html
├── package.json / vite.config.ts / tsconfig.json
├── public/assets/          # 模型、音频、贴图
└── src/
    ├── main.ts             # 入口：渲染器、SceneManager 启动
    ├── core/
    │   ├── SceneManager.ts
    │   ├── SaveSystem.ts
    │   ├── AssetLoader.ts
    │   └── AudioManager.ts # M4
    ├── scenes/
    │   ├── MapScene.ts     # 地图 + 10 山 + 点亮状态 + 点击拾取
    │   └── MeditationScene.ts # 主题模板 + 老师 + 机位
    ├── systems/
    │   ├── DialogueSystem.ts
    │   ├── MeditationSession.ts
    │   └── LightingRitual.ts
    ├── ui/                 # HUD、对话框、完成页（HTML 操作）
    └── data/
        ├── hills.json      # 10 山配置表（附录 D）
        ├── dialogues/      # 每位老师一份 JSON
        └── meditations/    # 每座山 5min/10min 引导脚本
```

### 7.3 数据驱动原则

- 10 座山的差异全部收敛到 `hills.json` + 对话/引导 JSON，代码零硬编码主题。
- 新增/调整内容只改数据文件，不改场景代码。

### 7.4 关键实现要点

- **点击拾取**：Raycaster 对山体组；悬停改变 emissive + 显示浮签。
- **点亮着色**：山体材质顶点色 × 状态系数（unlit 降饱和），灯光对象按状态 `visible`。
- **相机**：地图用 OrbitControls（带阻尼、限位）；冥想场景用脚本化相机动画（lerp 补间）。
- **时间轴**：冥想 cue 用统一 `Timeline` 小工具（注册 t 秒回调），暂停/恢复支持。

## 8. 里程碑与验收标准

### M1 — 地图原型（技术验证）

- Vite + TS + Three.js 工程脚手架；`npm run dev` 可预览。
- 程序化低模地形 + 10 座山按 `hills.json` 落位。
- OrbitControls（限位/阻尼/自动旋转）；悬停高亮 + 山名浮签；点击进入空冥想场景并可返回。
- 点亮状态两种视觉（灰暗/暖光）+ 手动调试切换；localStorage 读写。
- **验收**：桌面浏览器流畅 60fps；点击每座山能进出；状态刷新后保留。

### M2 — 单场景闭环（垂直切片）

- 青竹林完整主题场景（地形装饰、配色、雾、三段机位）。
- 老师低模 + 程序化待机；点击触发对话（打字机 + 选项）。
- 冥想会话：5/10 分钟选择、文字引导 cue 时间轴、计时圆环、呼吸动画、中断确认。
- 点亮演出（场景内 + 地图常驻）+ 完成页统计。
- **验收**：从地图进青竹林 → 对话 → 冥想 5 分钟 → 点亮 → 返回地图山变亮，全程无断点；中途退出不计完成。

### M3 — 内容铺开

- 其余 9 个主题场景（模板换肤 + 装饰差异）。
- 10 位老师对话剧本（各 2~4 轮 + 选项）与 5/10 分钟引导词（各一套）。
- 进度 x/10、重复冥想累计统计。
- **验收**：10 座山全部可完整走完闭环；文案逐山校对通过。

### M4 — 体验打磨

- Howler.js 环境音 ×10 + 交互音效 + **引导语 TTS**（与文字 cue 同步）。
- 粒子/光效增强；点亮全局完成演出（10/10 彩蛋 + 古寺回访对话）。
- 移动端触摸适配评估与基础支持；性能优化（合批、实例化）。
- **验收**：音视频同步无明显延迟；中端设备帧率稳定；10/10 演出触发正确。

> 里程碑的任务级拆解放入 `docs/task/`，技术方案细节放入 `docs/tdd/`（后续开发时另行创建）。

## 9. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| 引导文案质量参差不齐 | 核心体验受损 | 统一文案模板（开场安顿→身体→呼吸→收尾），逐主题打磨，M3 集中校对 |
| 低模素材风格不统一 | 画面违和 | 锁定 1~2 个素材包为主，缺失件程序化自制；统一调色板后处理 |
| 冥想计时在后台标签页不准 | 会话中断 | 用 `performance.now()` 差值计时而非累加；页面隐藏时暂停并提示 |
| 单次开发范围失控 | 进度拖延 | 严格按 M1→M2 垂直切片推进，M2 通过后再铺内容 |
| TTS 音色不佳（M4） | 沉浸感下降 | 预留真人录音替换接口；TTS 仅作为默认方案 |
