## 1. 环境音循环（验收标准第 1 条）

逐山进山（点击首次手势后音频解锁），核对每山环境音主题与渐起渐弱：

| 山 | 环境音主题 | 听感要点 |
|---|---|---|
| bamboo 青竹林 | 竹林风声 | 风吹竹叶沙沙 |
| snowpeak 白雪顶 | 山风 | 高海拔空旷风鸣 |
| lakeside 镜湖畔 | 湖水 | 轻拍岸边水声 |
| starfield 星空原 | 夜虫 | 夜晚虫鸣+微风 |
| desert 寂静沙漠 | 风沙 | 低哑风沙流动 |
| sakura 樱花坡 | 春风 | 柔和暖风 |
| mistvalley 雾谷 | 雾鸣 | 空旷谷地低鸣 |
| grassland 风草原 | 草浪 | 风吹草浪起伏 |
| hotspring 温泉谷 | 温泉 | 水汽/气泡轻响 |
| temple 古寺 | 寺钟 | 远钟+庭院氛围 |

每山通用项：

- [ ] 进山后环境音约 2s 渐起；回地图约 1.2s 渐弱，无突兀中断
- [ ] 循环无明显接缝（同一环境音至少听满 2 个循环，约 44s）
- [ ] 点亮演出有颂钵音效（lit）、完成页有 complete 音效、UI 操作有确认音
- [ ] 环境音与该山主题匹配（对照上表）

## 2. 语音引导与同步（第 2 条）

- [ ] 冥想中引导语有沉稳男声语音，与 HUD 文字 cue 同步出现（`?debug` 加速抽查，偏差 < 300ms；对照 `public/audio/voice/manifest.json` 的文本）
- [ ] 抽查 10 山 × 2 档（5/10 分钟）各至少 1 条 cue；语音时长不同导致的时间轴微调自然（每条语音后有 ≥3s 静默，不抢拍）
- [ ] 静音开关同时控制环境音与语音（静音后全部无声，取消静音恢复）
- [ ] 回退验证：临时把某山一条语音 mp3 改名/移出，该 cue 回退纯文字且不阻塞流程；恢复后正常

## 3. 10/10 终局演出（第 3 条，B2b）

- [ ] 用 `?debug` 面板点亮全部 10 山，回地图触发终局演出：镜头拉全景 → 十山环绕"熄灭-重亮"波浪 → 逐山钵音 → 终局贺词面板（#finale）
- [ ] 贺词面板「前往古寺」触发彩蛋对话（古寺终章彩蛋文案）；「自由漫游」可继续在地图浏览
- [ ] 演出只播一次：再次回地图不重复触发（刷新页面后也不重播）
- [ ] 10/10 后任一山时长面板出现「不限时」选项：正计时、无结束演出、Esc 退出不计入进度统计

## 4. 视觉增强抽查（任务 4.4）

- [ ] 已点亮的山在地图上常驻萤火（多山点亮时共存不杂乱）
- [ ] 点亮演出光柱有纵向流光与呼吸微闪（对比 M3 更细腻）
- [ ] `?perf=high/medium/low` 三档切换：粒子密度明显分级（1 / 0.6 / 0.35），画面其余表现一致

## 5. 移动端（第 4 条关联，任务 4.6）

可用浏览器 DevTools 设备模拟先行筛查，真机复核为遗留项：

- [ ] 触屏（或模拟）单指旋转视角、双指缩放正常（OrbitControls 原生触摸）
- [ ] 窄屏（≤720px）HUD / 对话 / 面板响应式布局不错位、可点击
- [ ] 触屏或窄屏自动降 medium 档（`?debug` 看 fps/draw calls 验证）
- [ ] 真机遗留项：iOS Safari 首次手势解锁后语音/环境音是否正常、帧率与触摸手感
